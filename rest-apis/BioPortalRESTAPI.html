BioPortal REST APIs for Ontology, Value Set, and Class Resources

BioPortal contains three types of resources relevant to CEDAR: ontologies, value sets, and classes.

Ontologies contain potentially large sets of classes arranged in a hierarchy.

Value sets contains a typically small (10-40) sets of classes in a flat list.

Classes are contained in both ontologies and value sets.

This document will cover the calls necessary for searching and browsing of ontologies, value sets, and classes. Call to create value sets and classes will be covered later (and are not yet available in BioPortal).

Resource Index

These are the resources.

  • /ontologies [GET]
  • /value_sets [GET]
  • /classes [GET]

Resources

/ontologies


Methods

GET

/ontologies

Get all BioPortal ontologies.

Example request URI(s):

  • http://data.bioontology.org/ontologies/

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing a description of all BioPortal ontologies.

    
                  [{
                  "administeredBy": [],
                  "acronym": "NCIT",
                  "name": "National Cancer Institute Thesaurus",
                  "summaryOnly": false,
                  "@id": "http://data.bioontology.org/ontologies/NCIT",
                  "@type": "http://data.bioontology.org/metadata/Ontology",
                  "links": {},
                  "@context": {}
                  },
                  {},
                  {}
                  ]

    For SOW3 the relevant fields are acronym, name, and @id.

    This call does not return any ontology metrics (such as the number of classes in the ontology), which we indicated may be worth presenting. We can skip this for the moment and only return the information returned above.

    Note that the list of ontologies in BioPortal changes infrequently (every few days at most) so caching the list of ontologies is an option.

  • 401 If calling user does not have permission to view the ontology
/ontologies/{acronym}/submissions

Get the details of a specific ontology.

Request parameters:

parameter value description

acronym

string

the acronym of the ontology

Example request URI(s):

  • http://data.bioontology.org/ontologies/NCIT/latest_submission

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing a description of an ontology.

    
                      {
                      "contact": [],
                      "ontology": {},
                      "hasOntologyLanguage": "OWL",
                      "released": "2015-05-06T18:07:23-07:00",
                      "creationDate": "2015-05-06T18:07:24-07:00",
                      "homepage": "http://ncicb.nci.nih.gov/core/EVS",
                      "publication": null,
                      "documentation": "http://nciterms.nci.nih.gov",
                      "version": "15.04d",
                      "description": "A vocabulary for clinical care, translational and basic research, and public
                      information and administrative activities.",
                      "status": "production",
                      "submissionId": 42,
                      "@id": "http://data.bioontology.org/ontologies/NCIT/submissions/42",
                      "@type": "http://data.bioontology.org/metadata/OntologySubmission",
                      "links": {},
                      "@context": {}
                      }

    For SOW3 the relevant fields are description, version, hasOntologyLanguage, and documentation.

    This call does not return any ontology metrics (such as the number of classes in the ontology), which we indicated may be worth presenting. We can skip this for the moment and only return the information returned above.

  • 401 If calling user does not have permission to view the ontology

Note that there is currently no way to query BioPortal ontologies by name. Since the set of ontologies is fixed, applications typically cache the JSON response from the call to get all ontologies and perform search within that.

/search?q={query}&ontologies=NLMVS&roots_only=true

Query value sets by name.

NOTE: The current implementation of this method incorrectly matches the values in a value set in addition to the name of the value sets themselves. The call will exhibit correct behaviour in a few week. The URI above will not be affected.

Request parameters:

parameter value description

query

string

a query string matching a value set name

Example request URI(s):

  • Get value sets about "hypertension": http://data.bioontology.org/search?q=hypertension&ontologies=NLMVS&roots_only=true

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing value sets.

    
                      {
                      "page": 1,
                      "pageCount": 4,
                      "prevPage": null,
                      "nextPage": 2,
                      "links": {
                      "nextPage":
                      "http://data.bioontology.org/search?q=hypertension&ontologies=NLMVS&roots_only=true&apikey=3bdf57dc-4d53-4ca1-b6c1-a1f1fe651ea9&page=2",
                      "prevPage": null
                      },
                      "collection": [
                      {
                      "prefLabel": "Essential Hypertension",
                      "obsolete": false,
                      "matchType": "prefLabel",
                      "@id": "http://purl.bioontology.org/ontology/NLMVS/2.16.840.1.113883.3.464.1003.104.12.1011",
                      "@type": "http://www.w3.org/2002/07/owl#Class",
                      "links": {},
                      "@context": {}
                      },
                      {},
                      {}
                      ]
                      }

    The search results are paginated. The pageCount field indicates the number of pages, the page field indicates the number of the current page, and the nextPage field indicates the number of the next page. The links field can be used to navigate between pages using the contained nextPage and prevPage fields.

    Descriptions of value sets matching the search are returned in an array called collection. The relevant fields are prefLabel, which is the text that should be displayed for the value set name and the @id field, which uniquely identifies the value set and that can be used to get more information about it using the method described below.

  • 401 If calling user does not have permission to view the ontology
/ontologies/NLMVS/classes/{id}?include=prefLabel,definition,properties

Get the details of a specific value set.

Request parameters:

parameter value description

id

string

the id of the value set

Example request URI(s):

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing details for a value set.

    
                      {
                      "prefLabel": "Bradycardia",
                      "definition": [],
                      "properties": {
                      "http://purl.bioontology.org/ontology/NLMVS/Version": [
                      "2.16.840.1.113883.3.526.2.411"
                      ],
                      "http://purl.bioontology.org/ontology/NLMVS/OID": [
                      "2.16.840.1.113883.3.526.3.412"
                      ],
                      "http://purl.bioontology.org/ontology/NLMVS/Definition": [
                      "Bradycardia"
                      ],
                      "http://purl.bioontology.org/ontology/NLMVS/VersionComment": [
                      "MU2 Update 2015-05-01"
                      ],
                      "http://www.w3.org/2004/02/skos/core#prefLabel": [
                      "Bradycardia"
                      ],
                      "http://www.w3.org/1999/02/22-rdf-syntax-ns#type": [
                      "http://www.w3.org/2004/02/skos/core#Concept",
                      "http://www.w3.org/2002/07/owl#NamedIndividual"
                      ]
                      },
                      "@id": "http://purl.bioontology.org/ontology/NLMVS/2.16.840.1.113883.3.464.1003.106.12.1001",
                      "@type": "http://www.w3.org/2002/07/owl#Class",
                      "links": {},
                      "@context": {}
                      }

    For SOW3 the relevant fields are prefLabel, which corresponds to the name of the value set, and all the fields inside "properties" that start with "http://purl.bioontology.org/ontology/NLMVS/". The relevant information for the value set "Bradycardia" is:

    • Value set name: Bradycardia
    • Version: 2.16.840.1.113883.3.526.2.411
    • OID: 2.16.840.1.113883.3.526.3.412
    • Definition: Bradycardia
    • VersionComment: MU2 Update 2015-05-01

  • 401 If calling user does not have permission to view the ontology

/classes


Methods

GET

/search?q={query}

Query all classes in BioPortal.

Query parameters:

parameter value description

query

string

a query string matching a class name

Example request URI(s):

  • http://data.bioontology.org/search?q=studies

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing a set of classes.

    
                        {
                        "page": 1,
                        "pageCount": 17,
                        "prevPage": null,
                        "nextPage": 2,
                        "links": {
                        "nextPage":
                        "http://data.bioontology.org/search/?q=studies&apikey=3bdf57dc-4d53-4ca1-b6c1-a1f1fe651ea9&page=2",
                        "prevPage": null
                        },
                        "collection": [
                        {
                        "prefLabel": "study",
                        "synonym": ["studies"],
                        "definition": ["a planned process that consists of parts"],
                        "obsolete": false,
                        "matchType": "synonym",
                        "@id": "http://purl.org/ddi/owl#study",
                        "@type": "http://www.w3.org/2002/07/owl#Class",
                        "links": {
                        "self":
                        "http://data.bioontology.org/ontologies/HUPSON/classes/http%3A%2F%2Fpurl.org%2Fddi%2Fowl%23study",
                        "ontology": "http://data.bioontology.org/ontologies/HUPSON"
                        },
                        "@context": {}
                        }
                        ]
                        }

    The search results are paginated. The pageCount field indicates the number of pages, the page field indicates the number of the current page, and the nextPage field indicates the number of the next page. The links field can be used to navigate between pages using the contained nextPage and prevPage fields.

    Descriptions of classes matching the search are returned in an array called collection. The relevant fields are prefLabel which is the text that should be displayed for the class name. The definition field may also be relevant. The @id field uniquely identifies the class.

    The links object contains two relevant fields. The self field contains the full BioPortal URL to get details class. This URL can be used to get detailed information on the class.

    The ontology field contains the URL of the entology containing the class. The last fragment of the name corresponds to the acronym of the ontology. This acronym can be used as the display name of the source ontology. It can also be used get full details on the ontology itself using the ontology REST routes.

  • 401 If calling user does not have permission to query the classes
/ontologies/{acronym}/classes/{id}

Get the details of a specific class.

Request parameters:

parameter value description

acronym

string

the acronym uniquely identifying an ontology

id

string

the id of the class

Example request URI(s):

  • http://data.bioontology.org/ontologies/HUPSON/classes/http%3A%2F%2Fpurl.org%2Fddi%2Fowl%23study

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing a description of a class.

    
                        {
                        "prefLabel": "study",
                        "synonym": [
                        "studies"
                        ],
                        "definition": [
                        "a planned process that consists of parts"
                        ],
                        "cui": [ ],
                        "semanticType": [ ],
                        "obsolete": false,
                        "@id": "http://purl.org/ddi/owl#study",
                        "@type": "http://www.w3.org/2002/07/owl#Class",
                        "links": {},
                        "@context": {}
                        }

    The prefLabel field can be used as the display name of the class. The definition field contains a textual description of the class. The @id field uniquely identifies the class.

  • 401 If calling user does not have permission to view the class
/ontologies/{acronym}/classes/{id}/tree

Get the minimally expanded tree hierarchy for a specific class.

Request parameters:

parameter value description

acronym

string

the acronym uniquely identifying an ontology

id

string

the id of the class

Example request URI(s):

  • http://data.bioontology.org/ontologies/SYN/classes/http%3A%2F%2Fncicb.nci.nih.gov%2Fxml%2Fowl%2FEVS%2FThesaurus.owl%23Nonclinical_Study_Type/tree

Available response representations:

  • 200 application/json [expand]

    Example

    An example of a response containing a description of a tree hierarchy for a class.

    
                        [
                        {
                        "prefLabel": "Manufacturer",
                        "hasChildren": true,
                        "children": [],
                        "obsolete": false,
                        "@id": "http://purl.bioontology.org/ontology/SYN#Manufacturer",
                        "@type": "http://www.w3.org/2002/07/owl#Class",
                        "links": {},
                        "@context": {}
                        },
                        {
                        "prefLabel": "Activity",
                        "hasChildren": true,
                        "children": [
                        {
                        "prefLabel": "Action",
                        "hasChildren": true,
                        "children": [ ],
                        "obsolete": false,
                        "@id": "http://ncicb.nci.nih.gov/xml/owl/EVS/Thesaurus.owl#Action",
                        "@type": "http://www.w3.org/2002/07/owl#Class",
                        "links": {},
                        "@context": {}
                        }
                        ]
                        }
                        ]

    This call will return a minimally-expanded tree hierarchy for a specific class. The returned array contains list of parent classes in the ontology tree.

    The prefLabelfield in each class object can be used to display the class name. The hasChildren field indicates if a class has children. The children field contains the child classes for that class if they are expanded. This field can be used to recurisvely construct an expanded tree.

  • 401 If calling user does not have permission to view the class hierarchy
